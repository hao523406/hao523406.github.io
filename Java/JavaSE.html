# 基本概述

  • Java是SUN(Stanford University Network，斯坦福大学网络公司 ) 1995年推出的一门高级编程语言。

  • Java是一个纯粹的面向对象的程序设计语言，它继承了C++语言面向对象技术的核心。Java舍弃了C语言中容易引起错误的指针（以引用取代）、运算符重载、多重继承（以接口取代）等特性，增加了垃圾回收器功能用于回收不再被引用的对象所占据的内存空间。JDK1.5又引入了泛型编程、枚举、不定长参数、自动装/拆箱。

  • Java之父：James Gosling。

  • 发展简史：

    • 1991年 Green项目，开发语言最初命名为Oak (橡树)
    • 1994年，开发组意识到Oak 非常适合于互联网
    • 1996年，发布JDK 1.0，约8.3万个网页应用Java技术来制作
    • 1997年，发布JDK 1.1，JavaOne会议召开，创当时全球同类会议规模之最
    • 1998年，发布JDK 1.2，同年发布企业平台J2EE
    • 1999年，Java分成J2SE、J2EE和J2ME，JSP/Servlet技术诞生
    • 2004年，发布里程碑式版本：JDK 1.5，为突出此版本的重要性，更名为JDK 5.0
    • 2005年，J2SE -> JavaSE，J2EE -> JavaEE，J2ME -> JavaME
    • 2009年，Oracle公司收购SUN，交易价格74亿美元
    • 2011年，发布JDK 7.0
    • 2014年，发布JDK 8.0，是继JDK 5.0以来变化最大的版本
    • 2017年，发布JDK 9.0，最大限度实现模块化
    • ......

  • Java技术体系平台

    • **Java SE **(Java Standard Edition)：标准版

      支持面向桌面级应用（如Windows下的应用程序）的Java平台，提供了完整的Java核 心API。

    • **Java EE **(Java Enterprise Edition)：企业版

      为开发企业环境下的应用程序提供的一套解决方案。该技术体系中包含的技术如Servlet 、Jsp等，

      主要针对于Web应用程序开发。

    • Java ME (Java Micro Edition)：小型版

      支持Java程序运行在移动终端上的平台，对Java API有所精简，并加入了针对移动终端的支持。

    • Java Card

      支持一些Java小程序（Applets）运行在小内存设备（如智能卡）上的平台。

# Java运行机制

  • Java特点：

    • 面向对象

      基本概念：类、对象。

      三大特性：封装、继承、多态。

    • 健壮性

      吸收了C/C++语言的优点，但去掉了其影响程序健壮性的部分（如指针、内存的申请与释放等），提供了一个相对安全的内存管理和访问机制。

    • 跨平台性

      通过Java语言编写的应用程序在不同的系统平台上都可以运行。

      因为有了JVM，同一个Java程序在不同的操作系统中都可以执行。实现了Java 程序的跨平台性。

  • Java核心机制：

    • Java虚拟机 (Java Virtal Machine)

      • JVM是一个虚拟的计算机，具有指令集并使用不同的存储区域。负责执行指令，管理数据、内存、寄存器。
      • 不同的平台，有不同的虚拟机 (JVM是不跨平台的)。只有该平台提供了对应的JVM，java程序才能在此平台上运行。
      • JVM屏蔽了底层运行平台的差别，实现了“一次编译，到处运行”。

    • 垃圾回收机制 (Garbage Collection)

      • 垃圾回收：回收不再使用的内存空间。
      • Java消除了程序员回收无用内存空间的责任。它提供一种系统级线程跟踪存储空间的分配情况。并在JVM空闲时，检查并释放那些可被释放的存储空间。
      • Java程序运行中自动进行垃圾回收，程序员无法精确控制和干预。因此，Java程序还是会出现内存泄漏和内存溢出的问题。

# Java环境搭建

JDK = JRE + 开发工具集(javac, java 等工具)

JRE = JVM + JavaSE标准类库

  • JDK (Java Development Kit)：Java开发工具包

    JDK是提供给Java开发人员使用的，其中包含了java的开发工具，也包括了JRE。

    所以安装了JDK，就不用在单独安装JRE了。

  • JRE (Java Runtime Environment)：Java运行环境

    包括Java虚拟机和Java程序所需的核心类库等。

    如果想要运行一个开发好的Java程序，只需要JRE即可。

  • 安装

    下载JDK，并安装或解压，确定已配置PATH。

    目的：方便在任意路径下使用 javac, java 等命令，不需要再进入jdk目录/bin下。

    • PATH

    为了方便后续的Java工具的使用，我们使用 JAVA_HOME 的方式来配置。

    • JAVA_HOME + PATH

    • 检验：java -version

# 标识符

  • 标识符：Java 对变量、方法和类等命名时，使用的字符序列称为标识符。

  • 标识符的规则：

    • 26个英文字母大小写、0到9、_ 、$ 组成。

    • 数字不能开头，不能包含空格。

    • 不能使用关键字和保留字。

    • Java严格区分大小写，长度无限制。

    • 可以使用汉字，但强烈不推荐使用 (包括汉字的拼音)。

  • 命名规范 (遵循标识符)：

    • 包 名：所有字母小写
    • 类、接口 名：首字母大写
    • 变量、方法 名：首字母小写
    • 常量 名：所有字母大写，且用**_**连接多个单词

# 变量

  • 变量：用于在内存中保存数据，数据可以在同一数据类型范围内不断变化。

    • Java中的变量：必须先定义，后使用。
    • 使用变量名来访问这块区域的数据。
    • 变量的作用域：定义所在的{ }内，只有在其作用域内才有效，同作用域内不能定义同名的变量。

  • 变量的定义与赋值：

    语法

    变量的定义(声明)：<数据类型> <变量名称>

    变量的赋值：<变量名称> = <值>

    变量的定义和赋值 (初始化)：<数据类型> `<变量名称> = <值>

    // 变量的定义
int num;

// 变量的赋值
num = 10;

// 变量的定义和赋值
int count = 5;

  • 变量的分类：对于每一种数据都定义了具体的数据类型，在内存中分配了不同大小的内存空间。

# 基本数据类型

  • 整数类型：byte、short、int、long

    • Java整型常量 默认：int。默认值：0。(long型常量必须后加l或L，默认值：0L)

  • 浮点类型：float、double

    • Java浮点型常量默认：double。默认值：0.0。(float型常量必须后加f或F，默认值：0.0F)
    • 表现形式：
      • 十进制数 形式 (5.12 、0.251 、.256)
      • 科学计数法 形式 (5.12e2 、515E2 、100E-2)

  • 字符类型：char

    • char类型表示”字符“，占用2字节，范围：0～65535，默认值：'u0000'。

    • Java所有字符使用Unicode编码，所以一个字符可以存储一个字母、一个汉字、一个数字字符 等。

    • char类型是可以运算的，对应其Unicode码来运算。

    • 表现形式：

      • 一对单引号括起来的单个字符。(如：char c = 'A')

      • Java使用**转义字符\**表示特殊字符。

        转义字符 说明
        \b 退格符
        \n 换行符
        \r 回车符
        \t 制表符
        \" 双引号
        \' 单引号
        \\ 反斜线

      • 直接使用 Unicode 值来表示字符型常量。(如：'\u000a'表示'\n'，\后面是十六进制)

  • 布尔类型：boolean

    • boolean类型用来判断逻辑条件，只有 true 和 false，默认值：false。
    • Java虚拟机中没有任何供boolean值专用的字节码指令，Java语言表达所操作的boolean值，在编译之后都使用java虚拟机中的int数据类型来代替：true用1表示，false用0表示。
      但是boolean数组编译之后会使用byte数组来代替。

# 基本数据类型的转换

  • 自动类型转换：容量小的类型 自动转换为 容量大的类型。

    • byte、short、char 之间不会相互转换，计算时首先转换为int类型。
    • boolean类型不能与其他基本数据类型运算。
    • 多种类型的数据混合运算时，Java首先会将所有数据转换为容量最大的数据类型，再运算。

  • 强制类型转换：强制将 容量大的类型 转换为 容量小的类型，可能造成精度降低或溢出。

    • 语法：(小类型)<大类型>
    • boolean类型不能转换为其他基本数据类型。

# 进制

  • 整数的表示方式：

    • 二进制(binary)：0b 或 0B 开头。
    • 八进制(octal)：0 开头。
    • 十进制(decimal)
    • 十六进制(hex)：0x 或 0X 开头。(A-F不区分大小写)

  • 二进制整数的形式：

    • 原码：一个二进制数，最高位是符号位 (整数0，负数1)。
    • 反码：负数即符号位1时，符号位不变，原码按位取反。
    • 补码：负数即符号位1时，符号位不变，反码+1。
    • 正数：原码、反码、补码 都相同。
    • 计算机底层都以补码的方式来存储数据。

# 3.程序流程控制

  • 三种基本流程结构：
    • 顺序结构
    • 分支结构
    • 循环结构

# 顺序结构

程序从上到下逐行地执行，中间没有任何判断和跳转。

int num2 = num1 + 2;	// 报错
int num1 = 12;

# 分支结构

  • if-else 结构

    if (条件表达式) {
    true：执行代码块
}

    if (条件表达式) {
    true：执行代码块1
} else {
	false：执行代码块2
}

    if (条件表达式1) {
    true：执行代码块1
} else if (条件表达式2) {
    true：执行代码块2
}else {
	false：执行代码块3
}

    条件表达式：结果必须是boolean类型

    多个条件，是从上到下，必经过 条件表达式1，条件表达式2，...... 到else。

  • switch-case 结构

    switch (表达式) {
    case 常量1:
        语句1;
        break;
        
    case 常量2:
        语句2;
        break;
    ......
    case 常量n:
        语句n;
        break;
        
    default:
        语句;
}
    • switch( 表达式 ) 的类型：
      char、byte、short、int、Character、Byte、Short、Integer、enum(jdk 5)、String(jdk 7)。
    • case字句的值：必须是常量、互不相同、switch(表达式)的类型，不能是变量或变量表达式。
    • break语句用来执行完一个case分支跳出switch语句块；如果没有break语句，程序会从匹配的case分支开始一直往下执行直到break语句或结尾。
    • default子句是可选的，位置任意。当没有匹配的case字句，会执行default分支。
    • switch本质上只支持int类型，数字类型编译后：自动转换为int；枚举编译后：使用ordinal()方法返回int类型；字符串编译后：使用hashCode()方法返回int类型，case的语句使用equals()比较。
    // 枚举 类
enum Stauts {
    SUCCESS,
    FIAL;
}

// switch使用枚举
Stauts stauts = Stauts.SUCCESS;
switch (stauts) {
    case SUCCESS:
        System.out.println("成功");;
        break;

    case FIAL:
        System.out.println("失败");;
        break;
}

    • 注意：case值不需要写枚举类，直接写枚举对象名。

    • 使用 switch-case 的，都可以改写为 if-else，反之不成立。

# 循环结构

  • 循环结构：

    • 初始化：可以定义多个变量，但必须同一类型，逗号分开。
    • 循环条件：boolean表达式，true时进入循环，false退出循环。
    • 迭代：可以多个变量更新，逗号分开。
    • 循环体

  • for 循环

    for (初始化; 循环条件; 迭代) {
    循环体
}

    可以只写循环体。

  • while 循环

    初始化
while (循环条件) {
    循环体
    迭代
}

    循环条件不能写false，编译报错。

  • do-while 循环

    初始化
do {
    循环体
    迭代
}while (循环条件)

    至少执行一次循环体。

# 特殊流程控制

  • break：终止某个语句块的执行。

    • 循环：跳出当前循环。(不是跳出所有循环)
    • switch：退出某个case分支。

  • continue：跳过一次循环，继续下一次循环。

    • continue语句只能用于 循环结构。
    • 出现多层嵌套时，continue语句可以通过标签指明要跳过的是哪一层循环。

  • return：结束一个方法。

break、continue、return 语句之后不能出现语句。

# 4. 数组

# 数组概念

  • 数组：多个相同类型数据按一定顺序排列的集合，并通过编号的方式对这些数据进行统一管理。

  • 数组本身是引用数据类型，数组的元素可以是 基本数据类型 和 引用数据类型。

  • 创建数组对象会在内存开辟一整块连续的空间，数组名引用这块连续空间的首地址。

  • 数组的长度一旦确定，就不能修改。

  • 通过索引(下标)的方式操作指定位置的元素，速度很快。

  • 数组工具类：java.util.Arrays

  • 数组常见异常：

    • java.lang.ArrayIndexOutOfBoundsException：数组索引越界异常。
    • java.lang.NullPointerException：空指针异常。

# 一维数组

  • 定义(声明) 与 初始化

    • 定义：

      // 定义：<元素类型>[] 变量名
int[] nums;
String[] names;
// 数组声明，不能指定长度，如：int nums[5];

    • 初始化

      • 静态初始化：定义同时赋值。(定义和赋值不能分开)

        // 初始化有多少个元素，就确定长度为多少。
int[] arr = new int[]{1, 3, 5};
// 简写
int[] arr = {1, 3, 5};

      • 动态初始化：定义、赋值 分开。初始化必须在中括号写上 [长度]，确定长度后，数组每个元素都是该数据类型的默认值 (引用数据类型默认：null)。

        int[] nums = new int[3];
nums[0] = 1;
nums[1] = 3;
// 或者
String[] names;
names = new String[5];
names[0] = "Java";
names[1]  "C++";

  • 使用

    • 获取元素：数组名[索引]，[ ]里：int类型 常量或表达式。

    • 数组元素的索引：从0开始 到 数组长度-1。

    • 数组的长度：属性length (int类型)。初始化一旦确定，长度不可变。

      int[] arr = {1, 2, 3, 4, 5};
// 获取元素
int num = arr[2];
// 元素赋值
arr[3] = 33;
// 获取长度
int len = arr.length;

# 多维数组

  • 初始化

    • 静态初始化

      int[][] arr = new int[][]{ {1,2,3}, {4,5,6}, {7,8,9} };
// 简写
int[][] arr = { {1,2,3}, {4,5,6}, {7,8,9} };

    • 动态初始化





       



      // 定义一个二维数组，有3个长度为2的二维数组。
int[][] arr = new int[3][2];

// 二维数组初始化只指定一维数组的长度，一维元素全部为null。
int[][] arr = new int[3][];

int[][] arr = new int[][3];		// 非法
    • 特殊写法：int[] x, y[]; // x是一维数组，y是二维数组。
    • Java中多维数组不必都是规则矩阵形式。

  • 使用

    • 获取元素：

      int[][] arr = new int[3][2];

// 一维元素：数组名[一维索引]； int[] e = arr[0];

// 二维元素：数组名[一维索引][二维索引]	
int ee = arr[0][1];
```

# 5. 面向对象

# 思想

  • 面向对象：将工程封装进对象，强调具备了功能得对象，以类/对象为最小单位，考虑谁来做。

    面向对象：OOP (Object Oriented Programming)

  • 三大特性：

    • 封装
    • 继承
    • 多态

# 类、对象

  • 类与对象：

    • 类 (Class)：对一类事物的描述，是抽象的、概念上的定义。
    • 对象 (Object)：实际存在的该类事物的每个个体，因而也称为实例(instance)。

  • 类的成员：

    • 属性：类中的成员变量 (准确来说，成员变量是字段。属性是对应字段的getter方法或setter方法)。
    • 行为：类中的成员方法。

  • 类的语法：

    [修饰符] class 类名 {
    成员变量
    成员方法
}

// 人 类
public class Person {
    String name;	// 成员变量：姓名
    
    // 成员方法：工作
    void doWork() {
        System.out.println(name+" 工作中...");
    }
}

  • 对象：

    • 创建：类名 对象名 = new 类名();

      Person p = new Person();	// 如果对象的成员变量未初始化，会用默认值来初始化。

      如果创建了一个类的多个对象，对于对象的属性，每个对象都拥有各自的一套副本，且互不干扰。

    • 使用：对象名.对象成员;

      // 获取对象的成员变量
String name = p.name;
// 对象的成员变量赋值
p.name = "Hao";
// 使用对象的成员方法
p.doWork();

    • 匿名对象：不将对象的引用地址赋值给一个变量。(就是这个对象没有对象名，没有变量引用)

      new Person().doWork();

      匿名对象只能使用一次。
